# ComunidadIFTS

Plataforma web de ComunidadIFTS con frontend Angular y backend PHP para mapa institucional, autenticacion, administracion, personalizacion del sitio, perfiles y bolsa de trabajo.

## Documentacion centralizada

La documentacion humana vigente del proyecto se organiza en la carpeta `docs`.

Esta estructura fue alineada con el estandar compartido definido en `../ESTANDAR_DOCUMENTACION_PROYECTOS.md`.

- `docs/README.md`
- `docs/estado-actual.md`
- `docs/QA_EJECUCION_16-ABRIL-2026.md`
- `docs/QA_ROLES_Y_BOLSA_TRABAJO.md`

La documentacion tecnica especifica del backend vive en `BackEnd/docs`.

## Punto de inicio recomendado

1. Leer `docs/README.md`.
2. Seguir con `docs/estado-actual.md` para entender arquitectura, rutas y modulos vigentes.
3. Ir a `docs/deploy.md` si el objetivo es despliegue.

## Regla de mantenimiento

La documentacion vigente debe actualizarse en `docs/` y no volver a dispersarse en markdown sueltos de la raiz.
